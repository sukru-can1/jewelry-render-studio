<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Server Action (createBatch) | Untrusted selection payload crosses here; validated by `createBatchSchema` (this plan) before any logic |
| domain DB keys -> recipe generator keys | Admin-editable strings cross into the hardcoded generator key space; `binding.ts` is the only sanctioned crossing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | createBatch selection payload | mitigate | `createBatchSchema` enum-validates keys, requires >=1 angle/metal/pass, caps array lengths (anti-automation, V5) |
| T-03-02 | Tampering | unsupported StoneType.key -> recipe material_map | mitigate | `resolveStoneMaterial` returns null for any non-generator-supported material; caller (03-02) rejects rather than emitting a material the worker can't honor (RESEARCH Pitfall 3) |
| T-03-03 | Denial of Service (cost) | client-computed job count | mitigate | `BATCH_LIMITS`/`countJobs` are a single shared source the server (03-02) recomputes against; client value is advisory only (BATCH-06) |
| T-03-SC | Tampering | npm/shadcn installs | accept | No new npm runtime deps this plan (RESEARCH Package Legitimacy Audit: none); shadcn components added in 03-03 are first-party copies |
</threat_model>
