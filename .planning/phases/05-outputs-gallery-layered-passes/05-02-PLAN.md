<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RunPod worker output → deriveLayerFromResult | Untrusted JSON from the worker becomes Layer fields |
| Webhook (at-least-once) → Layer write | Duplicate completion callbacks |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | duplicate webhook → duplicate Layer | mitigate | upsert on `Layer.jobId @unique` (idempotent); guarded by the existing terminal-status check upstream |
| T-05-02 | Tampering / Injection | worker output → Layer.url | mitigate | store only the blob PATHNAME, never the worker-supplied public `image_url`; skip on missing pathname (no throw) |
| T-05-03 | Information Disclosure | leaking public blob URL as delivery href | mitigate | Layer.url is the private pathname; delivery is via the /api/file proxy only (SEC-02) |
</threat_model>
