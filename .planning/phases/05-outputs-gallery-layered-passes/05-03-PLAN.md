## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/file & zip route | Authed-but-untrusted query params (pathname, name, scope) |
| zip route → private Blob | Server re-reads private blobs; client never touches Blob |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Spoofing | unauthed download/zip | mitigate | `requireSession()` fail-closed 401 first line of both routes |
| T-05-05 | Tampering | header injection via `name` filename | mitigate | sanitize Content-Disposition filename (strip CR/LF + quotes) in both routes |
| T-05-06 | Elevation / IDOR | arbitrary batch zip | mitigate | zip scope derived from DB rows loaded by params.id, not caller pathnames; single-tenant posture documented (A5) |
| T-05-07 | Information Disclosure | public-URL leakage of private assets | mitigate | only `get(...,{access:'private'})`; no signed/public URL construction; blob-guard source test |
| T-05-08 | Denial of Service | huge batch zip overruns 60s | accept/mitigate | stream (no buffer); prefer per-group scope; cap/paginate flagged for very large sets |
</threat_model>
