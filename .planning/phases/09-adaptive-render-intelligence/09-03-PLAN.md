<objective>
Make the loop human-in-the-loop and never-silent. Three surfaces:

1. The "Optimize with AI" opt-in toggle on the batch builder (default OFF; sets
   `optimizeWithAi` in the createBatch payload; hidden/disabled with a note when the
   feature is not configured) — INTEL-05 opt-in.
2. The per-job intel panel on the batch detail monitor: the preview thumbnail, the
   D1–D8 score bars, the flags, the applied knob deltas, the rationale, the loop's
   reached decision, and the distinct ESCALATED "needs human" state with its reason
   (09-AI-SPEC §7.2).
3. The operator Accept / Reject / Override Server Action (auth-first, IDOR-scoped,
   logs to `Job.intel.operatorAction`) plus the DB-only intel read.

Purpose: satisfy the phase's non-negotiable human-in-the-loop constraint — the operator
ALWAYS sees the AI's scores + reasoning + applied knob deltas and can accept/reject;
escalations surface why; nothing ships silently. The UI holds no authority — the Server
Action re-checks auth + scope.
Output: 1 builder edit (toggle), 1 intel panel component, 1 monitor wire, 1 operator-action
Server Action, 1 DB-only read, 2 RED test files driven GREEN.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → operator-action Server Action | Untrusted {jobId, action, overrideIteration} crosses the action boundary; validated + IDOR-scoped |
| operator UI → ship decision | The accept/override decision controls what ships; logged, never silent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-10 | Spoofing | unauth operator action | mitigate | requireSession() first line of applyIntelDecision, fail-closed — asserted in intel-operator-actions |
| T-09-11 | Elevation/IDOR | acting on another project's job | mitigate | the job is loaded WITH its batch and scope-checked before any write; unknown job → no write |
| T-09-12 | Repudiation | a silent / unattributed ship | mitigate | every operator decision logs operatorAction {action,userId,at}; the panel always shows scores+reasoning (never silent) — INTEL-05 |
| T-09-13 | Information Disclosure | preview thumbnail leaked publicly | mitigate | thumbnail via privateUrl → the auth-gated /api/file proxy (the browser session is verified), never a public URL |
</threat_model>
