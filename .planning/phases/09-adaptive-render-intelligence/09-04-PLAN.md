<objective>
Build the calibration gate that must be GREEN before auto-correct is trusted — the
reference-dataset harness from 09-AI-SPEC §5.7. It is the empirical proof that the
vision judge agrees with the QA lead before the loop is allowed to auto-adjust knobs
unsupervised.

1. A 12–20 case labelled reference set (`fixtures/labels.json`) covering known-good
   (ring99) + known-bad (zanessa), all three metals, all four angles, every DOMAIN
   failure mode, and one empty/black adversarial — each with human 1–5 dimension
   scores, the expected delta SIGN per non-accept case, and the expected hard-gate firing.
2. Pure calibration helpers (`lib/intelligence/calibration.ts`): within-±1, delta
   sign-agreement, the judge↔human agreement score, and the `autoCorrectTrusted` gate.
3. The runner (`run-eval.ts`) + a vitest wrapper (`calibration.test.ts`): score the set
   (verdicts cached so CI doesn't burn vision calls), assert ±1 per dimension, sign-agreement
   on every non-accept case (the "raise brightness to fix milky" anti-pattern catcher), and
   all three hard gates fire on the bad set; compute agreement.
4. Gate the loop: `decideLoop` returns `autoCorrect` ONLY when `autoCorrectTrusted` is true;
   otherwise the verdict is surfaced RECOMMEND-ONLY (operator applies/declines, no auto
   re-preview). Default recommend-only until calibration proves ≥0.7.

Purpose: this is the trust gate (INTEL-06). The system MUST NOT auto-correct unsupervised
until the judge demonstrably agrees with the human ≥0.7; below that it recommends only.
Output: 1 labelled fixture set, 1 pure calibration lib, 1 runner, 1 vitest wrapper, 1
loop.ts gate edit.
</objective>
