<objective>
Build the impure half of the loop: the vision scorer and the async state-machine
orchestration, wired onto the EXISTING dispatch/webhook/reconcile machinery without a
second code path.

1. `lib/intelligence/preview-image.ts` — fetch the PRIVATE preview blob bytes
   (`get(pathname,{access:'private'})`, NOT via the auth-gated /api/file proxy),
   downscale to ~768px PNG with sharp, return a base64 data URL (09-AI-RESEARCH §2).
2. `lib/intelligence/analyze-preview.ts` — the single vision call: `generateObject`
   against `visionVerdictSchema` (from 09-01) with `imageDetail:"low"` and the
   `generateText`+safe-parse fallback ladder lifted from `ai-classify.ts` (INTEL-02).
3. `lib/intelligence/sweep.ts` — the ANALYZING cron sweep (shaped like `reconcileJobs`):
   claim ANALYZING jobs optimistically, run `analyzePreview`, apply `decideLoop`, and
   re-dispatch a preview / queue the final / escalate — all guarded `updateMany`,
   budget-capped, idempotent (INTEL-04).
4. Wire-in: the webhook's `completed` branch flips an intelligence-preview job to
   `ANALYZING` (fast, no vision call); the reconcile cron route also runs the sweep;
   `createBatch` honors `optimizeWithAi` (opt-in) to mark preview jobs + seed PREVIEW_QUEUED.

Purpose: make the loop run end-to-end across multiple async GPU renders using the proven
async pattern — preview/final are ordinary Jobs, the webhook stays fast, the slow
gpt-5.5-pro vision call runs on the cron sweep (not a 60s request path), and every
transition is an idempotent guarded updateMany recovered by the existing reconcile cron.
Output: 3 new intelligence libs, 1 webhook hook, 1 createBatch opt-in branch, 1 cron-route
wire, 5 RED test files driven GREEN.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| private Blob → vision model | Worker-rendered preview bytes (trusted origin) downscaled and sent to OpenAI as a data URL — never a public/signed URL |
| OpenAI model → loop state | The vision verdict crosses into the state machine; schema-validated (09-01) before any decision |
| RunPod webhook → intel state | The completion callback flips intel state; must be idempotent (at-least-once delivery) |
| cron → GPU re-dispatch | The sweep can enqueue more GPU renders; bounded by the cost cap (G8) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-05 | Information Disclosure | preview blob leaked via a public URL to the model | mitigate | previewDataUrl reads get(pathname,{access:'private'}) and base64-inlines it; never /api/file, never public — intel-db-only + intel-preview-image assert it |
| T-09-06 | DoS | a duplicate webhook / sweep tick double-dispatches GPU renders | mitigate | every transition is a guarded updateMany on the expected prior intelState (count===1 wins) — duplicate matches zero rows; cost cap G8 bounds total renders |
| T-09-07 | Tampering | the sweep hand-builds a recipe, bypassing the quality source (G10) | mitigate | sweep emits recipes ONLY via buildEnterpriseRecipe(profileOverrides); intel-db-only source-guards against hand-built recipe keys |
| T-09-08 | Elevation | loop runs when the operator/admin disabled it | mitigate | G9 kill-switch — OPENAI_API_KEY absent OR ADAPTIVE_INTELLIGENCE_ENABLED=false OR per-batch optimizeWithAi false → bypass; asserted in intel-sweep + intel-webhook-hook |
| T-09-09 | DoS | the slow vision call stalls the webhook 200 → RunPod retry storm | mitigate | the webhook ONLY flips state (fast); the vision call runs on the cron sweep, not the webhook — asserted in intel-webhook-hook (no analyze import in the webhook path) |
| T-09-SC | Tampering | npm installs | accept | no new dependencies — ai@^6, @ai-sdk/openai@^3, zod, sharp all already present (package.json verified); no install task this phase |
