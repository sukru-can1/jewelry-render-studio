<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator session → admin route | Privilege boundary; must 403 server-side regardless of hidden nav |
| client payload → user create/update | Untrusted input validated before any Prisma write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-RBAC | Elevation | admin user routes + pages | mitigate | `requireRole("Admin")` first line on every route AND page; rbac-enforce test proves Operator→403 |
| T-1-AUTH | Spoofing | created user credentials | mitigate | Temp password bcrypt(12) hashed; passwordHash never returned in list responses |
| T-1-INPUT | Tampering | user create/update payloads | mitigate | zod validation (email/role/password) before Prisma; Prisma parameterized (no raw SQL) |
| T-1-DISCLOSE | Info | user list response | mitigate | Select excludes passwordHash; only id/email/role/disabled/createdAt returned |
</threat_model>
