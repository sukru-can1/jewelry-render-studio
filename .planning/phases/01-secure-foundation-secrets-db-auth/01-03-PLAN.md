<objective>
Build the auth + RBAC + route-gating core: the split edge-safe `auth.config.ts` + Node `auth.ts` (Credentials
provider that bcrypt-checks the seeded User and issues a JWT cookie), the `requireRole()` server-side boundary,
the NextAuth route handler, deny-by-default middleware, and the shared-secret RunPod webhook scaffold. This is
the auth half of the Walking Skeleton — after this, the seeded Admin can authenticate and every route is locked.

Purpose: AUTH-01 (login + persistent cookie), AUTH-02 (logout via signOut), AUTH-03 (role in session),
SEC-03 (deny-by-default), SEC-04 (webhook secret). AUTH-04/05 enforcement is exercised in Plan 06 (admin routes);
the `requireRole` boundary itself is built and unit-tested here. AUTH-01/AUTH-02 get a dedicated integration test
(`test/auth-login.test.ts`) exercising authorize() + the jwt/session role shape + signOut cookie clearing.
Output: lib/auth/{auth.config.ts,auth.ts,rbac.ts}, app/api/auth/[...nextauth]/route.ts, middleware.ts,
app/api/webhooks/runpod/route.ts, types/next-auth.d.ts, auth-login/RBAC/deny/webhook tests.

NOTE: This plan depends only on 01-01 (it needs the generated Prisma client + User model + typed env, not a live
DB). The auth-login integration test mocks the Prisma `user.findUnique` so it runs without a live DB; live-login
E2E in the browser remains a manual check in Plan 05.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → app/API | Untrusted requests; deny-by-default unless authenticated session present |
| Operator session → Admin action | Privilege boundary enforced server-side, not by UI |
| RunPod → webhook | Machine-to-machine; no user session, authenticated by shared secret |
| edge middleware → Node handlers | Prisma/bcrypt must not cross into the edge runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-AUTH | Spoofing | Credentials login / session | mitigate | Auth.js v5 Credentials + bcrypt(12) compare; encrypted JWT HTTP-only cookie; generic error (no user enumeration); authorize() unit-tested for valid/bad/disabled |
| T-1-RBAC | Elevation | Operator→Admin escalation | mitigate | `requireRole("Admin")` server-side boundary (403); unit-tested; UI hiding is not the gate |
| T-1-AUTHZ | Elevation | deny-by-default bypass | mitigate | middleware matcher denies all but allowlist (/login, /api/auth, webhook); authorized callback returns false by default |
| T-1-WEBHOOK | Spoofing/Tampering | RunPod webhook | mitigate | `crypto.timingSafeEqual` against `RUNPOD_WEBHOOK_SECRET`; verified in handler not middleware |
| T-1-EDGE | Tampering | middleware edge runtime | mitigate | Split config — middleware imports only auth.config.ts; source-text test asserts no Prisma/bcrypt import |
</threat_model>
