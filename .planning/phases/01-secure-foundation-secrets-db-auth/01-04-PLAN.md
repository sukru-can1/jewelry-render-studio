## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → upload-token route | Untrusted; must prove session before a Blob write token is minted |
| caller → private asset | Untrusted; auth verified in the proxy handler next to get(), never middleware-only |
| working tree → remote | Leaked live secrets must be rotated and absent from tracked files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-BLOB | Tampering/Info | upload token + private delivery | mitigate | `requireSession()` in onBeforeGenerateToken; private store + auth-gated `get()` proxy; auth next to get() not middleware |
| T-1-SEC01 | Spoofing/Elevation | leaked RunPod/Blob secrets | mitigate | Rotate (dashboard); old key 401s; secrets only via typed env; `git grep` proves none in tree |
| T-1-SIGNEDURL | Info Disclosure | private asset model | mitigate | No signed-URL scheme (doesn't exist for private blobs); proxy is the only delivery path |
| T-1-SC | Tampering | @vercel/blob v2 upgrade | mitigate | Official Vercel package, audited in RESEARCH; v2.4.x required for private storage |
</threat_model>
