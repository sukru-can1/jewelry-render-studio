<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → inspection actions | untrusted productId |
| RunPod worker → private model blob | unauthenticated requests.get must still reach the model |
| worker → Blob sidecar | worker writes material_inventory.json with access=BLOB_ACCESS |
| worker output → app | inventory JSON (object/material names) is untrusted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Spoofing | inspection actions | mitigate | requireSession() first line on startInspection + pollInspection |
| T-02-13 | Info Disclosure | worker model fetch | mitigate | workerModelUrl mints a worker-readable signed URL only at dispatch (decision #1); not persisted/rendered as a public link |
| T-02-14 | Tampering/XSS | inventory object/material names | mitigate | names treated as opaque text; React escapes; mono text render; no dangerouslySetInnerHTML |
| T-02-15 | Info Disclosure | thumbnail/model in detail page | mitigate | delivered via authed /api/file proxy only, never result.url |
| T-02-16 | DoS | runaway poll loop | accept | poll is on-demand/interval-bounded for a <1min job; no per-request RunPod fan-out across products (Phase-4 webhook supersedes) |
| T-02-21 | Info Disclosure | inventory sidecar leaked via public URL | mitigate | BLOB_ACCESS=private so the worker writes the sidecar privately (user_setup); pollInspection reads it via @vercel/blob get(inventory_key, access:'private') by pathname — never via a public inventory_url |
| T-02-SC | Tampering | shadcn tabs/collapsible/accordion | mitigate | official shadcn registry only (first-party source copy) |
</threat_model>
