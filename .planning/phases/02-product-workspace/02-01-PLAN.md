<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → upload-token route | untrusted caller requests a Blob write token |
| RunPod worker → Blob | worker fetches the model by URL with no app session |
| seed/migration → live DB | schema + domain rows applied to Railway Postgres |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | app/api/blob/upload onBeforeGenerateToken | mitigate | requireSession() first line (already present); token mints only for a session |
| T-02-02 | Info Disclosure | model blob via public token | mitigate | add access:'private' to the token (this plan's fix); deliver later only via /api/file proxy |
| T-02-03 | Info Disclosure | worker model fetch (workerModelUrl) | mitigate | mint a short-lived/non-guessable worker-readable URL only at dispatch; never persist/render it as a public link; Phase-8 follow-up if fallback used |
| T-02-04 | Tampering | inventory JSON from worker (object/material names) | mitigate | parseInventory treats names as opaque strings; downstream React escapes; never dangerouslySetInnerHTML |
| T-02-SC | Tampering | npm/shadcn installs | mitigate | no new npm deps in this plan (RESEARCH Package Legitimacy Audit: none); shadcn blocks are first-party source copies |
</threat_model>
