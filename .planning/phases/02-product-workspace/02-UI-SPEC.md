# Phase 2 — UI Design Contract (Product Workspace)

> Visual and interaction contract for Phase 2. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Inheritance note (READ FIRST):** Phase 1's UI-SPEC (`01-UI-SPEC.md`, status: approved) is the **global design system** — token architecture (primitive → semantic → component), color (teal accent, no purple; dark default + light), Geist Sans/Mono typography, 8pt spacing, elevation/motion, the app shell (240/56px sidebar + 56px topbar), and the seed component library. **This Phase 2 spec does NOT redefine any of that.** It (1) specifies the FIVE new Phase 2 surfaces, (2) declares the few NET-NEW components those surfaces require (model dropzone, material-inventory viewer, group-assignment table, hex/swatch editor, number-field grid), and (3) reuses every Phase 1 token, component, status-pill vocabulary, and copy tone unchanged. Where a value is "inherited," the executor reads it from Phase 1, not from here.
>
> **Scale-forward note:** these surfaces are the upstream of Batch Builder (P3), Jobs (P4), Gallery (P5), Compositing (P6). The product card, status-pill vocabulary, the `alloycolour/diamond/stone2/stone3` group identity, and the inspection→assignment data model defined here are consumed directly by P3's matrix builder. Keep them stable.

## Spacing Scale

**Inherited unchanged from Phase 1** (8-point scale: xs 4 / sm 8 / md 16 / lg 24 / xl 32 / 2xl 48 / 3xl 64; Tailwind `1/2/4/6/8/12/16`). Layout constants (240/56 sidebar, 56 topbar, 1280 content max-width, 24px gutters, 8px card radius) inherited.

| Token | Value | Usage | Phase-2 application |
|-------|-------|-------|---------------------|
| xs | 4px | inline gaps | swatch-to-hex gap, group-pill inner padding, object→material chevron gap |

## Typography

**Inherited unchanged from Phase 1.** Four roles, two weights (400 / 600), tabular-nums on all tables and numeric fields.

| Role | Size | Weight | Line Height | Phase-2 usage |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 | 1.5 | product names, descriptions, inventory cells, form values |
| Label | 12px | 600 | 1.4 | table headers, field labels, group captions, status-pill text (uppercase 0.04em) |
| Heading | 20px | 600 | 1.3 | page titles ("Products", "New product"), product-detail title, dialog titles, section headers |
| Display | 28px | 600 | 1.2 | not used in P2 (reserved for login/hero) |

- **Mono (Geist Mono, 400, same sizes):** object names (`band_metal`, `center_diamond`, `Round_5`), material-slot names, Principled BSDF numerics, model file name + size (`ring99.glb · 47.2 MB`), camera az/el/focal/fstop, metal hex codes, quality sample counts. Mono is the signal that a value is machine-derived (from inspection) vs. operator-authored.
- Min interactive text 12px (Label) — group-meaning helper text and format hints sit at Body 14px, never smaller.

## Color

**Inherited unchanged from Phase 1** — dark default + light, teal accent (NOT purple), 60/30/10 discipline, full semantic + status vocabulary. P2 introduces **no new hues**; it consumes the existing status-pill set for product and inspection lifecycle states, and the existing chart-free neutral surfaces.

| Role | Value (dark / light) | Phase-2 usage |
|------|----------------------|---------------|
| Dominant (60%) | `#0A0A0A` / `#FFFFFF` | workspace background behind product grid + detail |
| Secondary (30%) | `#141414` card · `#1F1F1F` border / `#FAFAFA` card · `#E4E4E7` border | product cards, inventory table surface, dropzone resting fill, assignment-table surface |
| Accent (10%) | `#14B8A6` / `#0D9488` | see "Accent reserved for" |
| Destructive | `#F87171` / `#DC2626` | remove-uploaded-model confirm; no other destructive action in P2 |
| Foreground | `#FAFAFA`/`#A1A1AA` muted · light `#18181B`/`#71717A` | text + secondary text |

**Status-pill vocabulary (inherited; mapped to P2 lifecycles — DO NOT invent new colors):**

| P2 state | Inherited token | Where shown |
|----------|-----------------|-------------|
| Product: **needs inspection** (model uploaded, no inventory) | `neutral` / idle | product card, product-detail header |
| Product: **inspecting** (RunPod inspect job queued/running) | `info` / running | product card (with subtle pulse), detail header |
| Product: **inspected — needs groups** (inventory ready, assignment incomplete) | `warning` | product card, detail header |
| Product: **ready** (all required groups assigned + saved) | `success` | product card, detail header |
| Product: **inspection failed** | `destructive` / failed | product card, detail header (+ Retry) |
| Object group chip: alloycolour | `neutral` | inventory row, assignment table |
| Object group chip: diamond | `accent` (teal) | inventory row — diamond is the hero/center group, the one accent use earned here |
| Object group chip: stone2 / stone3 | `info` / `warning` respectively | distinguishable, color-blind-safe per Phase 1 ordering |
| Object group chip: **unassigned** | `neutral` outline (dashed) | inventory row before assignment |

**Accent reserved for (P2 — extends Phase 1's list, does not broaden it):**
- The single primary button per view ("New product", "Upload model", "Inspect materials", "Save groups", "Save changes").
- The active sidebar **Products** item (inherited shell behavior).
- Keyboard focus rings (inherited 2px teal).
- The **diamond** group chip (the center-stone group) — the one domain-semantic accent, chosen because diamond is the catalog hero and stays scarce (one group of four).
- **Dropzone active/drag-over state**: 2px teal dashed border + faint teal tint — a momentary accent on the drag interaction only, not the resting state.

Accent stays scarce: product "ready" uses `success` green, not teal; the metals editor swatches render the literal seeded metal hex (white/yellow/red gold), not accent.

## Phase 2 Surfaces

Each surface names its loading / empty / error / in-progress / success states (state-completeness is the inherited bar).

### 1. Products list — `/products` (the Products nav item, replacing the Phase 1 placeholder)
- **Header:** Heading "Products" + count (mono) on the left; primary button **"New product"** on the right.
- **Body:** responsive grid of **Product cards** (3-up at ≥1024px, 2-up ≥640px, 1-up below), each with thumbnail / name / mono filename / status pill / relative updated-at. Whole card click = reopen (→ product detail). Sort: most-recently-updated first.
- **States:** **loading** = 6 skeleton cards (inherited Skeleton); **empty** = centered empty-state ("No products yet" + body + [New product]); **error** = inline error card + Retry; per-card the status pill communicates lifecycle (needs inspection / inspecting / needs groups / ready / failed). No destructive action on this list in P2.

### 2. New product + model upload — `/products/new` (or dialog from list)
- **Form:** single-column. Field: **Product name** (text, required, inline-validated). Then the **Model dropzone**.
- **Dropzone:** accepts **GLB / FBX / BLEND / OBJ / STL**; uses the **authenticated direct-to-Blob client upload** (the Phase 1-hardened `blob/upload` token route now requires auth). Shows accepted formats + a **large-file note ("models up to ~50 MB; large files may take a moment")**. Live `progress` bar + mono % during upload; Cancel ghost.
- **States:** idle / drag-over / uploading / success / error (see Model dropzone states). **Validation:** wrong extension → "That file type isn't supported. Use GLB, FBX, BLEND, OBJ, or STL."; over size → "That model is larger than the 50 MB limit." Primary button **"Create product"** is disabled until name present + upload succeeded. On success → redirect to product detail with status **needs inspection**.

### 3. Product detail — `/products/[id]`
- **Header:** Heading = product name + status pill; secondary actions (Replace model, Rename) in a `…` menu; mono model metadata line (filename · size · format · uploaded-at).
- **Tabs:** **Overview** (model metadata + thumbnail) · **Materials** (inspection + inventory) · **Groups** (assignment).
- **Inspect action:** primary button **"Inspect materials"** dispatches the RunPod `inspect_materials` job. While running, show inline status (`queued` → `running` → `done` / `failed`) using inherited status pills + an info banner ("Inspecting materials — this usually takes under a minute. You can leave this page; we'll keep the result.").
- **Material inventory viewer** (on done): the **Material-inventory viewer** component populated from `inspect_materials.py` output (`objects[]` with `name`, `type`, `material_slots[]`, `bounds.max_dimension`; `materials[]` with `principled{}`). Collapsible per object; BSDF values mono/tabular; Base Color shown as a small swatch + RGBA mono.
- **States:** **not yet inspected** = empty-state with [Inspect materials]; **queued/running** = info pill + disabled re-inspect + skeleton inventory; **done** = inventory table; **failed** = destructive banner with the worker error summary (truncated) + **Retry inspection**; **stale** (model replaced after a prior inspection) = warning banner "This model changed since the last inspection. Re-inspect to refresh." + [Re-inspect].

### 4. Object → group assignment — Product detail "Groups" tab
- **The core of P2.** Lists every detected object (from inventory) as a **Group-assignment row**: object name (mono) + token hint + a group picker (alloycolour / diamond / stone2 / stone3 / **unassigned**).
- **Token-assist** (the **Token-assist bar**): object_signature substrings drive suggestions — `*metal*`/`*band*`/`*prong*` → alloycolour, `*center*`/`*solitaire*`/`*diamond*` → diamond, `*round_*`/`*side*` → stone2/stone3. Each suggestion is a dotted-teal hint the operator Accepts (never silently applied). A bulk action assigns all matching a token at once, with a preview count and toast undo.
- **Group-meaning affordance:** the **Group-meaning helper** explains each group and that **the saved assignment drives which objects are rendered or held out per pass (PROD-04)** — the bridge to Batch Builder.
- **Save:** **sticky save bar** (inherited Form pattern) appears when dirty: **"Save groups"** primary + "Discard" ghost. On save → product status recomputes (→ **ready** when the required groups are covered, else **needs groups**).
- **States:** **empty/partial** = unassigned rows shown with dashed neutral chips + a top hint "Assign each object to a group, then save."; **suggested** = token hints visible; **dirty** = save bar shown; **saving** = save-bar button loading; **saved** = success toast ("Groups saved.") + status pill updates; **incomplete-on-save** = non-blocking inline note listing still-unassigned objects (allowed — operator may intentionally leave decorative objects unassigned/other), with a warning that unassigned objects won't appear in any pass.

### 5. Domain settings EDIT (Admin) — `/admin/settings` (upgrades the Phase 1 read-only view; DATA-04)
- Upgrades Phase 1's read-only domain-settings view to **editable forms**. Admin-only (server-enforced per AUTH-05; nav item already Admin-gated). Operators deep-linking get the inherited calm 403 surface.
- **Sections (tabs or stacked, sticky save bar when dirty):**
  - **Camera views (4):** rows view1–4, each a **Number-field grid**: az / el / focal(mm) / fstop (mono). Seed values from DATA-03.
  - **Metals (3):** white / yellow / red — editable label + **Hex + swatch editor** (24px swatch + mono hex).
  - **Stone types:** editable list (cut × size × quality presets) — add/edit/remove rows; mono identifiers.
  - **Quality presets:** preview 64 / medium 256 / high 512 / ultra — editable mono sample counts; default resolution 1920×1920 shown (mono).
- **Validation:** focal > 0; fstop within 0.7–32; el within −90..90; az within −180..180; hex must be 6-digit; sample count a positive integer. Inline field errors (inherited Input error state).
- **States:** **loading** = skeleton tables; **error** = inline + Retry; **dirty** = sticky save bar ("Save changes" primary + "Discard"); **saving** = button loading; **success** = toast "Changes saved." with a quiet note that **changes apply to new batches** (not retroactive); **save failed** = "Couldn't save changes. Try again."
